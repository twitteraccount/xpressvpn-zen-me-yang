### ExpressVPN 协议

互联网协议决定了如何在网络上分发数据包。我们喜欢将数据包视为邮件，将协议视为邮递员可以采取的不同路线。当然，他们可以穿过树林更快到达目的地，但他们可能会在途中遇到狼。另一方面，如果他们只走主要高速公路，邮件将安全到达，但可能比穿过树林晚。

这个比喻也适用于互联网协议，因为安全的协议意味着更安全（尽管速度较慢）的 VPN 服务器。 ExpressVPN 提供以下 VPN 协议：

#### L2TP/IPsec

第 2 层隧道协议与 IPsec（见上文）配合使用，可创建非常安全的 VPN 客户端。L2TP 自行创建隧道，而 IPSec 负责加密和通道安全。它还可确保数据的完整性不会受到损害。

#### IKEv2

Internet Key Exchange 版本 2 是一种广泛使用的 VPN 协议，可在您断开互联网连接后自动重新建立与 VPN 的连接。对于喜欢在 Wi-Fi 和移动热点之间切换的人来说，这尤其有用，这是在公共场所使用互联网时非常常见的操作。

#### IPsec

IP 安全可创建安全的互联网连接，通常与其他协议一起使用以提高安全性（例如 IKEv2 和 L2TP）。IPsec 采用以下两种模式之一：传输模式（仅加密数据包消息本身）或隧道模式（加密整个数据包）。

#### OpenVPN

OpenVPN 是 VPN 中最受欢迎的协议，这是有原因的。它提供了安全性和速度的完美结合。此外，OpenVPN 提供两种不同的版本：

- **UDP：** UDP，即用户数据报协议，发送称为数据报的消息，不需要虚拟电路来传输数据。UDP 的优点是它需要的带宽较低，从而延迟较少。它的缺点是某些数据包可能会丢失或乱序。
- **TCP：**传输控制协议由七层组成，用于传输数据，并通过将数据分解为更小的数据包来确保数据到达正确的位置。它与 UDP 完美结合，因为它可以确保数据以正确的顺序重新组合。

UDP 和 TCP 就像是父母为旅行打包行李。虽然你的爸爸可能会更快地打包好你的东西，但你的妈妈会确保所有东西安全有序地到达目的地。

#### 光路协议

最后，ExpressVPN 提供其自己的专有协议 Lightway。Lightway 摒弃了非必要的过时功能。这意味着速度更快。同时，它使用 WolfSSL 进行加密，确保数据安全。

ExpressVPN 在其移动和桌面应用程序上广泛实施 Lightway，这是理所当然的，因为它是该品牌最稳定的协议之一。它也很轻量，这使其成为智能手机和笔记本电脑用户的良好协议。

不确定哪种协议适合您的具体情况？默认情况下，ExpressVPN 会在您登录时自动选择最佳协议。

### 可信网络服务

ExpressVPN 的一项高级功能是其 TrustedNetwork 服务。这项功能只有一些顶级 VPN 提供，也是 ExpressVPN 帮助开创的。但它是什么呢？好吧，TrustedNetwork 是 ExpressVPN 的说法，它拥有基于 RAM 的服务器或完全在随机存取存储器上运行的服务器。

服务器基本上就是计算机，传统上，VPN 服务器使用硬盘驱动器 (HDD) 来存储和运行其 VPN 操作系统。虽然信誉良好的无日志 VPN 的 HDD 服务器可以很好地保护敏感数据不被记录，但它们会在数据被清除后存储残留数据。不幸的是，残留数据可以恢复。3 如果政府扣押 VPN 的服务器，或者旧硬盘驱动器被拍卖（这是服务器场的常见做法），那么这就很危险了。

相比之下，RAM 每次重新启动时都会完全清除数据。这就是 RAM 被称为临时内存的原因。没有数据残留，VPN 操作系统必须从头开始安装。这可以保护服务器网络和用户数据免受内部攻击（例如流氓员工在服务器上安装后门）、政府和警察突袭以及基于 HDD 的服务器容易受到的其他威胁。

目前，ExpressVPN 的所有物理服务器都是基于 RAM 的。

### WebRTC 泄漏测试

![expressvpn-vpn-评论](https://www.security.org/app/uploads/2020/03/ExpressVPN-WebRTC-Leak-Test-screenshot.png)

![expressvpn-vpn-review](https://www.security.org/app/uploads/2020/03/ExpressVPN-WebRTC-Leak-Test-screenshot.png)

ExpressVPN WebRTC 泄漏测试。来自 ExpressVPN 网站的截图。

WebRTC，又称为 Web 实时通信，是一组标准化技术，允许 Web 浏览器直接相互通信。这意味着它们不必通过中间服务器。WebRTC 可提高速度，尤其是视频聊天、直播和文件传输。听起来很棒，对吧？

可能的安全问题源于 WebRTC 需要您的私人 IP 地址。

ISP（即互联网服务提供商）会发放公共 IP 地址。这些地址是全球唯一的，您的 ISP 只能为每个设备（通常是路由器）分配一个。公共 IP 地址允许用户直接访问互联网。

另一方面，路由器会向网络中的每个设备分配私有 IP 地址。大多数情况下，计算机、智能手机和平板电脑都会使用它们。VPN 会向您分配一个唯一的私有 IP 地址，这样就没有人可以跟踪您的在线活动。

[由于 WebRTC 使用您的私有 IP 地址，如果您使用 VPN，则测试 WebRTC 泄漏](https://www.security.org/vpn/webrtc-leak/)非常重要。泄漏可能会让您的地址被窥探。请记住，Firefox、Chrome、Opera 和 Microsoft Edge 都默认使用 WebRTC。

与 DNS 泄漏一样，ExpressVPN 在其网站上有一个 WebRTC 泄漏测试，而且使用起来非常简单。我们确保我们的 VPN 已打开，然后单击按钮以检测泄漏。

幸运的是，我们的 ExpressVPN 保护了我们免受 WebRTC 泄漏的影响。我们拥有快速的连接，同时又不危及我们的私有 IP 地址的安全。

## ExpressVPN 与 NordVPN

NordVPN 是另一款领先的 VPN，与 ExpressVPN 有很多相似之处。 ExpressVPN 在 160 个国家/地区拥有超过 2,000 台服务器，而当我们[评测 NordVPN](https://www.security.org/vpn/nordvpn/review/)时，我们发现它在 62 个国家/地区拥有多达 5,246 台服务器。不过，在做出决定之前，请务必查看其服务器位置。

此外，NordVPN 和 ExpressVPN 都不属于国际监控联盟，也不受数据保留法的约束，因为它们分别位于英属维尔京群岛和巴拿马。

#### 功能比较

| 特征 | ExpressVPN | NordVPN |
| --- | --- | --- |
| 日志数据 | 不 | 不 |
| 终止开关 | 是的 | 是的 |
| 隧道 | 分裂 | 不 |
| Netflix | 是的 | 是的 |
| 下载种子 | 是的 | 是的 |
| IP 地址 | 匿名，定期轮换 | [静态](https://www.security.org/vpn/static-ip-address/)，与其他 NordVPN 用户共享 |

这两种 VPN 都不会记录数据，包括会话的时间或持续时间、IP 地址、使用的服务器、下载的文件和访问的网站。两者都有终止开关，这是一项必不可少的安全功能，我们可以使用这两种 VPN 从 Netflix 流式传输并下载种子文件。

与 ExpressVPN 不同，NordVPN 不提供拆分隧道，因此我们无法同时访问私有网络和公共网络。虽然这两种 VPN 都提供匿名 IP 地址，但 NordVPN 每次都给我们相同的 IP 地址，这对安全性不利。与其他 NordVPN 用户共享我们的 IP 地址使我们更难被追踪。

如果您想要一个专用 IP 地址，您可以花一点钱购买一个。我们更喜欢 ExpressVPN 定期轮换其 IP 地址的方式，因为这会让黑客更难追踪我们。

现在我们已经比较了 NordVPN 和 ExpressVPN 的功能，让我们来谈谈它们的性能。虽然 ExpressVPN 在我们的 Windows 计算机上速度更快，但 NordVPN 在我们的 Mac 上运行得更好。这两个系统都没有任何 DNS 或 WebRTC 泄漏。

总体而言，这两个都是 VPN 的首选，但我们建议 Windows 用户使用 ExpressVPN，而 Mac 用户使用 NordVPN。
